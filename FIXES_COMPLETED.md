# Karna VLM Audit Fixes - Completed

All findings from `docs/AUDIT.md` have been addressed. All 88 tests pass.

## ✅ Critical Fixes (Training Blockers)

### 1. Collator-Bridge Interaction Fixed
**File**: `src/karna_vlm/data/collators.py`

**Problem**: Collator called `self.bridge(vision_out)` with `torch.no_grad()`, preventing gradient flow during training.

**Solution**:
- Created new `VLMTrainingCollator` that returns `pixel_values` (raw preprocessed images) instead of running bridge
- Added `prepare_training_batch()` method to `VLMModel` that handles vision encoding → bridge → packing with gradients enabled
- Training batch preparation now happens inside model forward, allowing gradients to flow through entire pipeline
- Original collator kept for inference (no breaking changes to inference API)

**Files changed**:
- `src/karna_vlm/data/collators.py` — added `VLMTrainingCollator`
- `src/karna_vlm/models/vlm_model.py` — added `prepare_training_batch()` method
- `src/karna_vlm/training/trainer.py` — uses new training collator

---

### 2. Forward Pass Label Alignment Fixed
**File**: `src/karna_vlm/models/vlm_model.py`

**Problem**: When using `text + image_features` path in `forward()`, external `labels` tensor wouldn't align with packed sequence length after prepending image tokens.

**Solution**:
- Added `forward_train()` method that accepts `images`, `prompts`, `responses`
- Method handles full pipeline: vision encoding → bridge → packing → label generation
- Labels are generated by packer from response text, ensuring perfect alignment with packed input
- Training now goes through `forward_train()` instead of generic `forward()`
- Generic `forward()` still works for inference

**Files changed**:
- `src/karna_vlm/models/vlm_model.py` — added `forward_train()` method
- `src/karna_vlm/models/prompt_packing/packer.py` — improved label masking
- `src/karna_vlm/training/trainer.py` — uses `forward_train()` in training loop

---

### 3. SafetyAction Enum Comparison Fixed
**File**: `src/karna_vlm/safety/policy.py`

**Problem**: `if rule.action.value > action.value` compared strings lexicographically, causing incorrect severity ordering ("warn" > "refuse" lexicographically).

**Solution**:
- Converted `SafetyAction` from `str` Enum to `IntEnum` with numeric severity values:
  ```python
  class SafetyAction(IntEnum):
      ALLOW = 0
      WARN = 1
      AUDIT = 2
      REFUSE = 3
  ```
- Now `REFUSE (3) > WARN (1)` correctly
- All tests updated to handle IntEnum (use `.name` for string comparisons)

**Files changed**:
- `src/karna_vlm/safety/policy.py` — SafetyAction → IntEnum
- `tests/test_safety.py` — updated assertions

## ✅ High Priority Fixes

### 4. Vision Encoder Preprocess Device Handling
**Files**: `src/karna_vlm/models/vision/siglip_encoder.py`, `clip_encoder.py`

**Problem**: `preprocess()` called `.to(device=device)` which would crash in DataLoader workers (workers can't access GPU).

**Solution**:
- `preprocess()` now keeps tensors on CPU
- Added explicit `.to(device)` calls in:
  - `VLMTrainingCollator` (moves pixel_values to device before returning batch)
  - `VLMModel.encode_image()` (moves to device before vision forward)
  - Inference paths (chat, generate)
- DataLoader workers now safe to use multi-worker parallelism

**Files changed**:
- `src/karna_vlm/models/vision/siglip_encoder.py`
- `src/karna_vlm/models/vision/clip_encoder.py`
- `src/karna_vlm/data/collators.py`
- `src/karna_vlm/models/vlm_model.py`

---

### 5. ChatSession Multi-Image Bug Fixed
**File**: `src/karna_vlm/inference/chat.py`

**Problem**: `_build_prompt()` added `<image>` token for every turn with an image, but `chat()` only passed `self.current_image` (single image) to generate, causing token/image count mismatch.

**Solution**:
- Chat now uses latest image only (single-image mode)
- `_build_prompt()` only adds `<image>` for the current turn's image
- Clears prompt history when new image is added (prevents stale `<image>` tokens)
- Added docstring clarifying single-image limitation

**Files changed**:
- `src/karna_vlm/inference/chat.py`

---

### 6. PromptPacker Label Construction Improved
**File**: `src/karna_vlm/models/prompt_packing/packer.py`

**Problem**: Label text handling was brittle — concatenated prompt+response then took last N tokens, causing misalignment when prompt length varied.

**Solution**:
- Now tokenizes prompt and response **separately**
- Creates proper label masking: -100 for prompt tokens, real IDs for response tokens
- Label tensor always matches `input_ids` length exactly
- Handles image token insertion correctly in label generation

**Files changed**:
- `src/karna_vlm/models/prompt_packing/packer.py`

---
